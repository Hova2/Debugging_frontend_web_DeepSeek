3. Panel Console La línea de comandos de tu navegador

El panel Console (o consola) es el mejor amigo del desarrollador. Es como tener una ventana directa al cerebro de JavaScript: puedes ver errores, imprimir mensajes, ejecutar código en vivo y hasta interactuar con la página.

💡 Analogía: Si el panel Elements es un quirófano para HTML/CSS, la consola es el monitor de signos vitales y el micrófono para hablarle directamente al código.
Consola del navegador (simulada)
> console.log("Hola desde el manual");
"Hola desde el manual"
> 2 + 2
4
> const usuario = { nombre: "Ana", edad: 30 }
▶ {nombre: "Ana", edad: 30}
>

⬆️ ¡Esta consola es interactiva! Prueba a escribir comandos y haz clic en "Ejecutar".

3.1 Abrir la consola

Formas de abrirla:

  • Atajo de teclado: F12 y luego haz clic en la pestaña "Console"
  • Atajo directo: Ctrl+Shift+J (Windows/Linux) / Cmd+Option+J (Mac) abre directamente la consola
  • Desde el inspector: Estando en Elements, pulsa Esc para abrir la consola en la parte inferior

¿Qué verás?

  • Errores en rojo (cuando algo falla)
  • Advertencias en amarillo (cosas que deberías revisar)
  • Mensajes de log (los que tú imprimas con console.log())
  • El resultado de cualquier expresión que escribas

⚡ Ejercicio 3.1: Abre tu consola real

1. Abre la consola real de tu navegador ahora mismo (F12 → pestaña Console, o Ctrl+Shift+J).

2. Escribe: console.log("Mi primer mensaje en la consola real") y pulsa Enter.

3. Escribe tu nombre entre comillas, por ejemplo "Ana", y pulsa Enter. ¿Qué devuelve?

4. Escribe 2 + 2 y luego "2" + 2. ¿Notas la diferencia?

💡 La consola evalúa cualquier expresión JavaScript y muestra el resultado. Las comillas hacen que sea texto.

3.2 console.log() y su familia

El método más famoso de la consola. Pero tiene varios primos que deberías conocer:

📝 console.log()

El clásico. Imprime cualquier cosa: textos, números, objetos.

console.log("Hola", 42, {x:1})

⚠️ console.warn()

Muestra una advertencia (en amarillo). Útil para avisos.

console.warn("Esto es una advertencia")

❌ console.error()

Muestra un error (en rojo). Ideal para fallos.

console.error("Algo salió mal")

📊 console.table()

Muestra arrays u objetos en formato tabla, mucho más legible.

console.table([{a:1, b:2}, {a:3, b:4}])

⏱️ console.time()

Mide cuánto tarda en ejecutarse un bloque de código.

console.time("bucle"); for(...){}; console.timeEnd("bucle")

🧹 console.clear()

Limpia la consola. Muy útil cuando está muy llena.

console.clear()

3.3 La consola como línea de comandos

La consola no solo muestra mensajes, también puedes ejecutar cualquier código JavaScript y modificar la página en vivo.

Comandos útiles en la consola:

  • document.title → Muestra el título de la página
  • document.body.style.background = "red" → Cambia el fondo
  • window.location.href → Muestra la URL actual
  • $0 → Hace referencia al elemento que tienes seleccionado en Elements
  • $_ → Resultado de la última operación
  • clear() → Limpia la consola

⚡ Ejercicio 3.3: Modifica la página desde la consola

En la consola real, prueba estos comandos (uno a uno):

  1. document.title = "Mi título modificado" → Verás cómo cambia el título de la pestaña
  2. document.body.style.background = "#0a1929" → El fondo se vuelve azul oscuro
  3. document.body.style.color = "white" → El texto se vuelve blanco
  4. document.querySelector("h1").style.color = "#64ffda" → El título principal cambia de color

💡 Estos cambios son temporales. Recarga la página para volver al estado original.

3.4 Leyendo errores como un profesional

Cuando algo falla, la consola muestra un mensaje de error. Aprender a leerlos es clave para debuggear rápido.

Uncaught TypeError: Cannot read property 'length' of undefined
at script.js:5:3

Partes de un error:

  • Tipo de error: TypeError, ReferenceError, SyntaxError, etc.
  • Mensaje: Cannot read property 'length' of undefined → Explica qué pasó
  • Archivo y línea: script.js:5:3 → Dónde ocurrió (archivo, línea, columna)

🔴 ReferenceError

console.log(variableNoDefinida);
Uncaught ReferenceError: variableNoDefinida is not defined

Significa: Usaste una variable que no existe.

🟠 TypeError

const num = 42;
num.toUpperCase();
Uncaught TypeError: num.toUpperCase is not a function

Significa: Intentaste usar un método que no pertenece a ese tipo de dato.

🟡 SyntaxError

console.log("Hola
Uncaught SyntaxError: Invalid or unexpected token

Significa: Error de escritura en el código (falta cerrar comillas, llaves, etc.)

⚪ undefined related

const obj = {};
obj.propiedad.otra
Uncaught TypeError: Cannot read property 'otra' of undefined

Significa: Intentaste acceder a una propiedad de algo que es undefined.

3.5 console.log() vs. debugger

El método más básico de debugging es llenar el código de console.log() para ver qué valor tienen las variables. Pero tiene limitaciones.

📝 console.log()

function suma(a, b) {
  console.log("a:", a);
  console.log("b:", b);
  let resultado = a + b;
  console.log("resultado:", resultado);
  return resultado;
}

Ventaja: Simple, rápido, no requiere aprender nada nuevo.

Desventaja: Hay que borrar los logs después, ensucia el código, no permite pausar la ejecución.

⏸️ debugger y breakpoints

function suma(a, b) {
  debugger; // Pausa aquí
  let resultado = a + b;
  return resultado;
}

Ventaja: Pausa la ejecución, puedes inspeccionar variables en ese momento, ver el call stack, etc.

Desventaja: Requiere abrir DevTools y conocer el panel Sources (lo veremos en el Capítulo 5).

Importante: Nunca dejes console.log() en código de producción. Son útiles para desarrollo, pero en producción pueden ralentizar y mostrar información interna.

⚡ Ejercicio 3.5: Encuentra el bug con console.log

La siguiente función tiene un bug. Usa console.log() en la consola para descubrirlo:

function calcularDescuento(precio, descuento) {
    let precioFinal = precio - descuento;
    return "El precio final es: " + precioFinal;
}

// Prueba:
calcularDescuento(100, 20);  // Esperamos 80
calcularDescuento(50, 10);   // Esperamos 40
🔍 Ver pista

El código parece correcto, pero... ¿y si el descuento es un porcentaje? Prueba calcularDescuento(100, "20") (con el descuento como string).

✅ Ver solución

El bug es que no se valida el tipo de dato. Si descuento es un string, JavaScript concatena en lugar de restar:

"100" - "20" → 80 (bien, JavaScript convierte)
pero "100" - "20%" → NaN
y si es string con texto, falla.

La solución sería convertir a número: precio - Number(descuento) o precio - parseFloat(descuento)

3.6 Trucos avanzados (que pocos conocen)

  • console.assert(condición, mensaje) → Solo imprime si la condición es falsa
  • console.count("etiqueta") → Cuenta cuántas veces se llama
  • console.group() y console.groupEnd() → Agrupa mensajes
  • console.trace() → Muestra la pila de llamadas (quién llamó a quién)
  • console.log({variable1, variable2}) → Atajo para imprimir variables con nombre

📌 Resumen del Capítulo 3

  • La consola se abre con F12 o Ctrl+Shift+J.
  • console.log(), warn(), error() y table() son tus herramientas básicas.
  • Puedes ejecutar cualquier código JavaScript directamente en la consola.
  • Los errores tienen tipo, mensaje y ubicación (archivo:línea).
  • console.log() es útil pero temporal; para debugging avanzado usaremos breakpoints.
  • Nunca dejes console.log() en producción.

🎯 Ejercicio final: Depura este código

El siguiente código tiene 3 errores (uno de cada tipo). Copia y pega en la consola real y encuentra los errores usando lo aprendido:

// Sistema de notas de estudiantes
const estudiantes = [
    { nombre: "Ana", nota: 85 },
    { nombre: "Luis", nota: 92 },
    { nombre: "Carlos", nota: 78 }
];

function calcularPromedio(estudiantes) {
    let suma = 0;
    for (let i = 0; i < estudiantes.lenght; i++) {  // Error 1
        suma += estudiantes[i].nota;
    }
    return suma / estudiantes.longitud;  // Error 2
}

function mostrarResultado(promedio) {
    console.log("El promedio es: " + promedi);  // Error 3
}

const promedio = calcularPromedio(estudiantes);
mostrarResultado(promedio);
🔍 Ver solución
  • Error 1 (Sintaxis/Runtime): estudiantes.lenght → debería ser estudiantes.length (propiedad incorrecta).
  • Error 2 (Runtime): estudiantes.longitud → no existe, debería ser estudiantes.length.
  • Error 3 (ReferenceError): promedi → la variable se llama promedio.
