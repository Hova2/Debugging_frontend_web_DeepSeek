1. Introducción al Debugging El arte de preguntarle al código

1.1 El primer bug de la historia

En 1947, en la Universidad de Harvard, un equipo de ingenieros trabajaba en la computadora Mark II. De repente, la máquina comenzó a dar errores. Al abrir los paneles, encontraron algo inesperado: una polilla (un moth) atrapada entre los relés.

Grace Hopper, pionera de la programación, pegó el insecto en el libro de registro con la nota: "First actual case of bug being found". Así nació el término "debugging".

📌 Dato curioso: Aunque Hopper popularizó el término, "bug" ya se usaba en ingeniería desde el siglo XIX para fallos mecánicos.

1.2 La mentalidad del detective

Debuggear no es solo "arreglar código". Es una forma de pensar. Adopta estos 3 principios:

🔍

Curiosidad

Pregúntate "¿por qué pasó esto?" en lugar de "no funciona". Cada error es una oportunidad.

⏳

Paciencia

Los bugs no se rinden fácil. Respira hondo y verifica, no adivines.

🧪

Método científico

Hipótesis → Experimento → Observación → Conclusión.

1.3 Los 3 tipos de errores

SINTAXIS missing ) after argument list

¿Qué es? Error de ortografía en el código. Al lenguaje no le gusta cómo escribiste algo.

❌ Incorrecto
console.log('Hola mundo'
✅ Correcto
console.log('Hola mundo');

🟢 Fáciles: El navegador te avisa con una línea roja.

LÓGICA El programa corre, pero hace otra cosa

¿Qué es? La receta está bien escrita, pero la instrucción es incorrecta.

❌ Queremos IVA
function precioConIva(precio) {
    return precio - 0.21;  // Resta en vez de sumar
}
✅ Correcto
function precioConIva(precio) {
    return precio * 1.21;
}

🟡 Tramposos: No hay mensajes de error, hay que descubrir que el resultado es incorrecto.

RUNTIME Uncaught TypeError: cannot read property 'x' of undefined

¿Qué es? El código es correcto, pero algo inesperado pasa al ejecutarse.

❌ Falla
const usuario = { nombre: 'Ana' };
console.log(usuario.direccion.calle);
✅ Seguro
const usuario = { nombre: 'Ana' };
console.log(usuario.direccion?.calle);

🔴 Explosivos: Rompen la aplicación y muestran error en consola.

  • 📝 Editor de código (VS Code): Donde escribes y corriges.
  • 🧠 Tu mente curiosa: La herramienta más importante.

📌 Resumen del capítulo

  • El primer bug fue una polilla real en 1947.
  • Debuggear requiere curiosidad, paciencia y método científico.
  • Errores de sintaxis: fáciles de detectar (faltan paréntesis, llaves).
  • Errores de lógica: el código funciona, pero mal (tramposos).
  • Errores de runtime: explotan al ejecutarse (impredecibles).
  • La consola del navegador es tu mejor aliada para empezar.
